### 개발시, 참고할 부분

1. **Lint**  
   **a. What is Lint?**

   > 소스 코드에서 문법적 오류, 스타일적 오류 등에 표시를 달아주는 행위

   **b. Linters**
   `ESLint`, `Prettier`

   - ESLint: 코드의 퀄리티 보장
   - prettier: 코드의 스타일을 통일

    **c. 읽어보면 좋은 자료**  
    https://helloinyong.tistory.com/325
